Swipe

  • jQuery Mobile: swipeleft and swiperight
  • swipe left to go right, swipe right to go left
    •     // swipe to the left
    $(element).on('swipeleft', function(){
        alert("swiped to the left")
    })

Transitions

  • What does a swipe mean if there's no transition associated with it?
  • Transitions are essential for good user experience.

$(element).on('swipeleft', function(){
    $.mobile.changePage(url, {transition: ‘slide’});
    // $.mobile.changePage(url, {transition: ‘slide’, reverse: true});
})

Pre-fetching

  • What does a transition mean if it's a poor transition?
  • Pre-fetching is essential for a good user experience!
  • 2 ways to declare it
    • Within the <a> 
      <a href="prefetchThisPage.html" data-prefetch> ... </a>
    • Within the js 
      $.mobile.loadPage( pageUrl, { showLoadMsg: false } );

All Together Now!

  • Swipe a transition to a pre-fetched page.
  • 
  // pre-fetch the page onload
  $(document).ready(function(){
    $.mobile.loadPage( nextUrl, { showLoadMsg: false } );
    // on swipeleft, progress to next url
    $(element).on('swipeleft', function(){
      $.mobile.changePage(nextUrl, {transition: 'slide'});
    })
  });

#navmenu { display: block; position: fixed; width: 300px; left: 0px; top: 0px; z-index: 0;  }

    pagebody.animate({
       left: "290px"

  • prototypes, not production-ready
  • overall, this creates issues for mobile, which is ironic
  • The problem is the styles are added, post-load, by javascript
  • Since I'm talking about optimization for mobile, this is necessary to discuss
  • The solution is to break out the styles of everything that is visible on the page when it loads and manually add the classes that are added post-load, it's bitchy, but recommended by jQuery Mobile
